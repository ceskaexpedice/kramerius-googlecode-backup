What do I find in a Digital library?

The Digital library contains valuable and endangered documents in a digital form from Memoriae Mundi Series Bohemica program (namely manuscripts, old prints and prospectively other documents) and from Kramerius program (older newspapers and journals and other valuable documents printed on an acidic paper). The access to the Digital library is limited to users with access privileges. Metadata, i.e. bibliographical and other descriptions of documents are accessible to all users as well as image document data exempt from Copyright Act protection. Image document data under the protection of Copyright Act are accessible only to users using PC located at the premises of the National Library CR.

What do I need to be able to work with a Digital library?

You need a web browser - Microsoft Internet Explorer 5.5 + is recommended or any other compatible browser. You need to install a plug-in module to be able to work with images in DjVu format. The advantage of this format is lower amount of transmitted data and ability to manipulate the image (maximize/minimize?) in the main browser using the aforementioned plug-in. We recommend you a screen resolution of 1024 x 728 pixels and higher. If you have a lower resolution problems might occur in manipulating the image files.

Access to the document pages follows the structure of these documents that is at first you are offered a bibliographical entry from which you proceed to sub layers of the documents (years, copies, particular pages). A level that you are in is always displayed on the menu located on the left side.

Searching documents

Searching allows easier discovery of the requested document. It runs over descriptive metadata for all types of documents including fulltext searching in page texts. The searching functionality is available on two places:
  • search field on the top of the welcome view and alphabet view
  • search field on the bottom of the left menu

Searching is applied to all types of documents. The results may be filtered with tabs that selects onle those results that are of selected type. If the search is performed for example during browsing the periodicals, the search will be automatically filtered to contain only periodicals. Searching is performed on these fields:

Periodicals:

  1. ISSN
  2. Main title, Sub-Title, Parallel title
  3. First and last name of the authors, First and last name of the contributors
  4. Publisher, Printer
  5. Date of publishing, Date of printing
  6. Shelf number
  7. Storage place
  8. Periodical volume number, Periodical item number
  9. Text content of the pages
Monographs:
  1. ISBN
  2. Main title, Sub-Title, Parallel title
  3. First and last name of the authors, First and last name of the contributors
  4. Publisher, Printer
  5. Date of publishing, Date of printing
  6. Shelf number
  7. Storage place
  8. Text content of the pages

Performing the search
The search field may be filled with words separated by space. These words are used for searching and are joined with AND logical operator by default. The mode may be changed by entering logical operators between the words. The expressions can be grouped with brackets.
AND - to search in AND mode, all the entered words have to be included (e.g. 'Česká AND republika')
OR - to search in OR mode, some of the entered words have to be included (e.g. 'Česká OR republika')
NOT - to avoid the following word (e.g. 'republika NOT Česká' will find only texts including 'republika' and not including 'Česká')
"exact phrase" - to search for exact phrase the words may be enclosed inside double apostrophes (e. g. "skákal pes přes")

Viewing and filtering the results
The search may be run by SEARCH button after entering the words into the search field. After that the search results page is diplayed with results sorted by the score. Every result contains a link to appropriate document. Search results may be filtered by tabs. The All tab will display all results, the Periodical tab will diplay only periodical results etc.

Advanced search
Advanced search may be used to enter specific value for each metadata field. For example you may search only for authors or titles. The search query of advanced search can be customized in results view and run again (you do not have to go back to advanced search view). If more than one field is filled with a value, these sub-queries are linked with AND operator which is default. If you need an OR operator, result query may be modified in a similar way:
author:(novák) title:(slovník)
to
author:(novák) OR title:(slovník)

Fulltext searching in documents content

The searching is performed on the content of the document texts. The fulltext results contain links that points directly to the page representation. The results may contain text snippets with highlighted occurences of the searched words. The page for advanced searching allows the user to specify similarity of words matching. The similarity may be set using a combo box with 10% to 100% values (example: when 70% is specified for searched word 'historie', results with 'historií', 'historik' či 'historika' words are returned).

Searching by date

This searching enables to search documents by year, month and day of its publication and printing and is accessible in advanced searching page. The form includes three combo boxes:
  • year
  • month
  • day
Select of the year field is mandatory, both month and day don't need to be selected. In case the day isn't selected all documents for the selected year and month are returned, similar behavior in case month isn't selected - all documents for the year are returned.

Selecting types of documents

This is the main screen of the system displayed after you have entered the system. This screen can also be displayed by using a link "Types of Documents" in the main application menu. This site contains search field and list of types of documents accessible in the system. At present it offers:
  • Periodicals
  • Monographs
  • Manuscripts and old prints - a link to an external application located at http://www.manuscriptorium.com/
You chose a requested document by clicking on its title. All selected document are displayed on all sites of lower level on an application status bar.
According to a selected type of document the number and content of sites of lower level change so that you get a suitable representation of the metadata structure describing the selected type of documents and allowing you well-arranged navigation (orientation) in the metadata structure for finding the requested document.
Numbers in brackets show number of titles and number of digitized documents (picture files) for given type.

Level 1 Periodical - Alphabet

This is the main screen for work with periodicals. It can be displayed by choosing "Periodical" item on " Documents type" site or by using a link "Alphabet" in the main application menu if you already are working with Periodicals type of documents.
This screen displays a list of selected capital letters including Czech letters with diacritical marks. Each letter is an active link to a site where you can find a list of titles of periodicals commencing with the selected letter. The selected letter is in all lower application levels displayed on the application status bar.
Numbers in brackets show number of titles and number of digitized documents (picture files) for given letter.

Level 2 Periodical - Title selection

This site allows you to choose a title. It can be displayed by choosing a letter on the "Alphabet" site or by using a link "Title selection" in the main application menu if you are already working with Periodicals type of documents and you have already selected some letter. This site contains a list in an alphabetical order of all titles of periodicals contained in the system which begin with a letter selected on the "Alphabet" site. Each title is a link to a "Title" site for the given title. The name of the selected title is in all lower application levels displayed on the application status bar.
Numbers in brackets show number of titles (allways 1) and number of digitized documents (picture files) for given title.

Level 3 Periodical - Title

This site can be displayed by choosing a title on the "Title selection" site or by using a link "Title" in the main application menu if you are just working with Periodicals type of documents and you have already selected some title. The site displays bibliographical description of the title and a list of all volumes of the title. Each volume serves as a link to a "Volume" site of the given volume.
Numbers in brackets show number of titles (allways 1) and number of digitized documents (picture files) for given volume.
The main application menu on this site contains the following items for more effective orientation within the selected title:
  • Volume - an active link to the "Volume" site which shall display the first volume of the current title.
  • Issue - an active link to the "Issue" site which shall display the first copy of the first volume of the current title.
  • Page - an active link to a "Page" site which shall display the first page of the first copy of the first volume of the current title or alternatively first page of the first volume of the current title if the first volume contains just pages and no copies.

Level 4 Periodical - Volume

This page can be displayed by choosing a volume on the "Title" site or using a link "Volume" in the main application menu.
It serves to display a bibliographical description of the current volume. Furthermore it displays a list of all issues of the current volume on condition that at least one such exists. Each of the displayed issues serves as a link to a "Issue" site for the given issue.
Numbers in brackets show number of titles (allways 1) and number of digitized documents (picture files) for given issue.
For some volumes, it is impossible during their digitalization to divide their pages into separate issues because they are for example numbered consecutively and bound together without distinct issue covers. If this is the case then the site contains a list of all pages of the current volume. Each of the displayed pages then serves as a link to a "Page" site displaying the selected page. Metadata can in this case also contain a description of the separate parts of the current volume (articles, tables, images). Each of the displayed parts serves as a link to a relevant "Part/Component" site.
The main application menu on this site contains the following items for more effective orientation within the selected title:
  • Title - an active link to a "Title" site which shall display a title into which the volume belongs.
  • Next to the "Volume" item you find an operating element for browsing previous and following volumes of the current title if they exist.
  • Issue - an active link to the "Issue" site which shall display the first issue of the first volume of the current title.
  • Page - an active link to a "Page" site which shall display the first page of the first issue of the first volume of the current title or alternatively first page of the first volume of the current title if the first volume contains just pages and no issues.

  • Next to the "Part" item you find an operating element for browsing previous and following parts of the current title or volume.

Level 8 Periodical - Page image

This site can be displayed by choosing a relevant link on a "Page" site or "Part/Component" site.
This site serves for displaying a digitalized document (image) of a certain page.
The image is displayed on a part of a page by using standard internet browser. However as DjVu is the primary format of the images in this application and this format is not commonly supported by ordinary browsers it is necessary to install this DjVu plug-in into the browser. If the document that should be displayed is not accessible for the public and the user does not access the system within the institution with access rights instead of the desired image the following notice appears: " Requested document not found".
The main application menu on this site contains the following items for more effective orientation within the selected title:
  • Title - an active link to a "Title" site which shall display a title into which the volume belongs.
  • Volume - an active link to a "Volume" site which shall display the volume the current copy belongs to.
  • Issue - an active link to a "Issue" site which shall display a issue the current page belongs to.
  • Page - an active link to a "Page" site which displays descriptive information about the page to which the current page document belongs.
  • Next to the "Page" item you find an operating element for browsing previous and following document (image) pages of the current volume or copy.

Level 5 Monograph - Physical unit

This site can be displayed by selection of any of the physical units on the "Monograph" site.
Physical unit means one of the physically independent parts of the monograph like appendices, maps, charts etc. A separate volume of the monograph for example individual volumes of an encyclopaedia etc. can also be labelled as a physical unit.
It enables displaying a bibliographical description of the selected physical unit and a list of all pages and internal component of the physical unit. Each of the listed pages serves as a link to a "Page" site for the given page. The same applies to a list of internal components where each item serves as a link to an "Internal component" site.
The main application menu on this site then contains the following items for more effective orientation within the selected monograph:
  • Monograph - an active link to a "Monograph" site which displays the monograph into which the active physical unit belongs.
  • Next to the "Physical unit" item there is a control element for browsing previous and next parts of the physical unit of the active monograph.
  • Page - an active link to a "Page" site which shall display the first page of the active physical unit of the monograph.

Level 6 Monograph - Internal component

This site can be displayed by selecting any of the internal components on "Monograph" or "Physical unit" sites.
Internal component means one of the logical parts (components) of the content (article, table, image, table of contents, etc.).
It enables displaying a bibliographical description of the selected component and it also contains a link to a site displaying the document (page image) where the component begins. If the document can not be displayed on the grounds of restrictions embodied in the Copyright Act the link will display metadata of this Page.
The main application menu on this site then contains the following items for more effective orientation within the selected monograph:
  • Monograph - an active link to a "Monograph" site which displays the monograph into which the active component belongs.
  • Physical unit - an active link to a "Physical unit" site which shall display the physical unit into which the active component belongs. This link is not present if the internal component is an inseparable part of the monograph.
  • Next to the "Internal component" item there is a control element for browsing previous and next parts of the active monograph or its physical unit.

Level 8 Monograph - Document page (Page image)

This page can be displayed by selecting an appropriate link on a "Page" site or on an "Internal component" site.
This site enables a display of a digitalized document (image) located on a certain page.
The image is displayed on a part of a site by standard means of an internet browser. As DjVu format is the primary format of the images and it is usually not supported by standard browsers, you may need to install DjVu plug-in module to be able to work with images in this format. If the document that should have been displayed is not accessible for the public and the user does not access the system from/within the institution with access rights a following notice instead of the desired image appears: "The requested document is not accessible".
The main application menu on this site then contains the following items for more effective orientation within the selected monograph:
  • Monograph - an active link to a "Monograph" site which displays the monograph into which the active page belongs.
  • Physical unit - an active link to a "Physical unit" site which shall display the physical unit into which the active page belongs. This link is not present if the page is an inseparable part of the monograph.
  • Page - an active link to a "Page" site which displays descriptive information about the page into which the active document page belongs
  • Next to the "Page" item there is a control element for browsing previous and next document pages (page images) of the active monograph or its physical unit.
